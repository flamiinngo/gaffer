## 1. What this is & what's at stake
**Gaffer** is an autonomous AI fantasy-football manager dApp built on **0G** for the **0G Zero
Cup Hackathon 2026** (~$17k prize pool). Users build an AI "gaffer", deploy it onchain, and it
**autonomously** plays the FIFA World Cup 2026 — picks its XI, captain and formation on 0G
Compute, is scored on real match stats (FPL rules), stores decisions on 0G Storage, and records
points onchain. The hook: the **Autonomy Multiplier** — the less you override it, the more you
earn (up to 3×).

**Status: we QUALIFIED TOP 32 of 200.** We are now improving the build for the **round-of-32
submission** (a few days). Competitors are strong (think Binance-tier teams). Bar: **world-class,
truly real (no mocks), every 0G layer genuinely used, verifiable from the homepage.**

**Winning thesis:** *Gaffer is the first **provably autonomous** AI agent you can own — it thinks
on 0G Compute, remembers on 0G Storage, proves itself on 0G DA, competes onchain, and you can
verify every decision yourself.*

## 4. Architecture
**Agent (`/agent`, Node ESM `.mjs`):**
- `populate-contest.mjs` — fetches a real WC matchday, builds the player pool, each of 3 gaffers
  picks an XI on **0G Compute**, scores it (FPL), stores each decision on **0G Storage**, records
  points **onchain**. Creates a NEW contest each run. (Now also captures team `colors`.)
- `export-leaderboard.mjs` — **auto-detects the latest fully-populated contest** from chain,
  downloads each decision from 0G Storage, and writes the frontend data files (below). Rewrites
  `featured.json` so data never drifts from the contest list.
- `gaffer.mjs` — CLI (init/status/deploy/run/override).
- Other `check-*.mjs` / `run-*.mjs` — proofs/utilities for each 0G layer.

**Frontend data files** (in `frontend/public/`, regenerated by the agent + committed daily):
- `proofs.json` — full verifiable record per agent (XI+bench with points, captain, reasoning,
  model, 0G Storage root, onchain tx/block). **Powers clickable profiles + live verification.**
- `leaderboard.json`, `gaffer-latest.json`, `featured.json`.

**Frontend (`/frontend`, Next.js app router):**
- Pages: `/` (home), `/contest`, `/contest/[id]`, `/gaffer/[address]` (agent profile),
  `/verify`, `/dashboard`, `/onboard`, `/developers`.
- Live verify API: `app/api/verify/[hash]/route.ts` — re-fetches the decision from 0G Storage by
  root hash, decodes the anchoring tx receipt, reads the manager's onchain record. Real, no mock.
- Key components: `pitch/Pitch.tsx` (FPL kits), `proof/ProofReceipt.tsx` (the "Verify on 0G"
  widget), `gaffer/GafferProfile.tsx`, `landing/HomeShowcase.tsx`, `contest/ContestBoard.tsx`,
  `onboard/OnboardFlow.tsx`.
- Libs: `lib/chain.ts`, `lib/abi.ts`, `lib/teamColors.ts`, `lib/onboardPreview.ts`,
  `lib/server/contract.ts`.

## 6b. Social / fun / ecosystem roadmap (to make it a real 0G project)
Vision: *a self-running economy of ownable, verifiable AI agents — competing in social leagues,
traded on a marketplace, every move provable on 0G.*

- **Social:** private leagues (invite link, private table — the FPL retention engine), head-to-head
  challenges with OG stakes, follow/spectate gaffers, shareable match-report cards (viral loop).
- **Fun:** chips/power-ups (Triple Captain, Bench Boost, Wildcard — onchain), a transfer market
  (prices move with form), live matchday points race, agent progression (titles/levels/reputation).
- **Real 0G economy:** tradeable agent NFTs (own/sell/rent a *trained* agent with a verifiable
  record), open agent SDK (bring your own model on 0G Compute), OG token sinks (entry/transfers/
  chips/challenges), reputation as a portable verifiable credential.
- **Recommended sequence:** Budget system → Private leagues (social flagship) → Shareable match
  cards (viral) → Chips (fun) → then the ownership/marketplace economy (post-hackathon).

## 7. How to run & test
```bash
# Frontend dev (already may be running on 3002)
cd frontend && npm run dev          # → http://localhost:3002  (NOT 3000)

# Regenerate live data from chain + 0G (needs .env with PRIVATE_KEY + SPORTAPI keys)
cd agent && node export-leaderboard.mjs        # safe: reads only, rewrites public/*.json
cd agent && node populate-contest.mjs          # writes onchain + creates a NEW contest (gas)

# Smoke test pages (replace ADDR with an address from proofs.json)
curl -s -o /dev/null -w '%{http_code}' http://localhost:3002/                 # home
curl -s http://localhost:3002/api/verify/<decisionRoot> | python -m json.tool # live proof
```
Test flow in browser: `/` → scroll to "The arena is live" → click a gaffer → scorecard → **Verify
on 0G** (watch 4 layers go green) → `/onboard` → Strategy step (drag sliders, watch XI rebuild).

**Gotchas:**
- Test on **3002**, not 3000.
- SportAPI keys are often **429** — populate/colour capture needs them un-throttled; they reset
  on a rolling window. The frontend uses committed `proofs.json`, so the UI works regardless.
- `export-leaderboard.mjs` auto-detects the latest contest — run it after any populate so the
  frontend data matches the chain.
- Privy "iframe failed to load" in the dev log is harmless auth noise.
